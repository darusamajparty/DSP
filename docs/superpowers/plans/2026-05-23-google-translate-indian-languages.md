# Google Translate Indian Languages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Hindi and major Indian-language support to the DSP Next.js site using Google Cloud Translation to create reviewed, stored translations instead of translating live on every request.

**Architecture:** Extract all public copy into typed locale dictionaries, generate draft non-English dictionaries with Google Cloud Translation, and serve localized routes such as `/hi`, `/mr`, and `/ta` with stable content. Keep API submission data unchanged and only localize labels, validation messages, page metadata, navigation, legal pages, and membership-card display text.

**Tech Stack:** Next.js 15 App Router, React 19, TypeScript, Google Cloud Translation Advanced v3, Node.js script tooling, existing Playwright QA script.

---

## Scope Decisions

- Default locale: `en`.
- Initial Indian-language launch set: Hindi `hi`, Marathi `mr`, Bengali `bn`, Gujarati `gu`, Punjabi `pa`, Tamil `ta`, Telugu `te`, Kannada `kn`, Malayalam `ml`, Urdu `ur`.
- Translation mode: Google Cloud Translation produces draft locale files during a controlled script run; production page loads do not call Google.
- Brand glossary: preserve `Daru Samaj Party`, `DSP`, `ACPs`, `Alcohol Consuming Persons`, hashtags, email addresses, URLs, membership IDs, and form field names submitted to the backend.
- Legal pages must be translated, but English remains the fallback if any locale key is missing.
- Membership form payload remains English-keyed because `app/api/members/route.ts` expects `name`, `email`, `phone`, `state`, `district`, `instagram`, and `photo`.
- Generated card visual text should localize only user-visible labels such as `STATE`, `DISTRICT`, `STATUS`, `ACTIVE`, `FOUNDING MEMBER`, and `MEMBERSHIP ID`; the ID format and user-entered values stay unchanged.

## External References

- Google Cloud Translation Advanced supports text and HTML translation and can retain HTML tags while translating only text nodes: https://cloud.google.com/translate/docs/advanced/translating-text-v3
- Google Cloud Translation language support includes Hindi and major Indian languages such as Bengali, Gujarati, Kannada, Malayalam, Marathi, Punjabi, Tamil, Telugu, and Urdu: https://cloud.google.com/translate/docs/languages
- Glossaries are available in Cloud Translation Advanced for terminology control: https://cloud.google.com/translate/docs/samples/translate-v3-translate-text-with-glossary

  en: "English",
  hi: "Hindi",

  bn: "Bengali",
  gu: "Gujarati",
  pa: "Punjabi",
  ta: "Tamil",
  te: "Telugu",
  kn: "Kannada",
  ml: "Malayalam",
  ur: "Urdu",
};

export const localeDirections: Record<Locale, "ltr" | "rtl"> = {
  en: "ltr",
  hi: "ltr",
  mr: "ltr",
  bn: "ltr",
  gu: "ltr",
  pa: "ltr",
  ta: "ltr",
  te: "ltr",
  kn: "ltr",
  ml: "ltr",
  ur: "rtl",
};

export function isLocale(value: string): value is Locale {
  return (locales as readonly string[]).includes(value);
}

export function localizedPath(locale: Locale, path = "") {
  const normalizedPath = path.startsWith("/") ? path : `/${path}`;
  if (locale === defaultLocale) return normalizedPath === "/" ? "/" : normalizedPath;
  return normalizedPath === "/" ? `/${locale}` : `/${locale}${normalizedPath}`;
}
```

type LocalePageProps = {
  params: Promise<{ locale: string }>;
};

export default async function LocaleHomePage({ params }: LocalePageProps) {

import {
  defaultLocale,
  localeNativeLabels,
  locales,
  localizedPath,
  type Locale,
} from "../lib/i18n/locales";

type LanguageSwitcherProps = {
  currentLocale: Locale;
  currentPath?: string;
  label: string;
};

export default function LanguageSwitcher({
  currentLocale,
  currentPath = "/",
  label,
}: LanguageSwitcherProps) {
  return (
    <label className="language-switcher">
      <span>{label}</span>
      <select
        defaultValue={currentLocale}
        onChange={(event) => {
          const nextLocale = event.target.value as Locale;
          window.location.href = localizedPath(
            nextLocale === defaultLocale ? defaultLocale : nextLocale,
            currentPath,
          );
        }}
      >
        {locales.map((locale) => (
          <option key={locale} value={locale}>
            {localeNativeLabels[locale]}
          </option>
        ))}
      </select>
    </label>
  );
}
```
